## 解决的问题

### 飞书发送本地媒体文件路径权限问题

**问题：** 直接发送桌面路径的图片报错 `LocalMediaAccessError: Local media path is not under an allowed directory`

**解决方案：** 
1. 先复制到工作目录的 `tmp/` 文件夹
2. 再用工作目录路径发送

```bash
cp /Users/yunsan/Desktop/a.png ~/.openclaw/workspace/tmp/a.png
```

**创建了 Skill：** `skills/media-workaround/SKILL.md` 详细记录了这个问题的处理流程。

## 配置说明

`~/.openclaw/openclaw.json` 中的 `mediaLocalRoots` 配置：
```json
"mediaLocalRoots": [
  "/tmp",
  "~/.openclaw/workspace",
  "/Users/yunsan/Desktop"
]
```

但 Desktop 路径可能有问题，`~/.openclaw/workspace` 是最可靠的。
